# Инструкция Для Reviewer

Вы получили blind-eval пакет для оценки двух ответов (`A` и `B`) на один и тот же пользовательский запрос.

## Что нужно сделать

1. Откройте `submission.csv`.
2. Для каждой строки поставьте оценки по шкале `1.0 .. 5.0`:
   - `clarity`
   - `empathy_appropriateness`
   - `usefulness`
   - `factuality`
3. Для каждой строки заполните structured review notes:
   - `robotic_repetition`: `none | minor | clear`
   - `unnatural_templating`: `none | minor | clear`
   - `language_mismatch`: `none | minor | clear`
   - `clarification_helpfulness`: `n_a | weak | mixed | strong`
   - `naturalness_feel`: `low | mixed | high`
4. Для `clarification_helpfulness` используйте `n_a`, если ответ не является clarification turn.
5. Если не уверены между двумя qualitative tags, выбирайте более мягкий вариант (`minor`, `mixed`), а не оставляйте поле пустым.
6. Structured notes не заменяют четыре численные оценки; заполняйте оба слоя.
7. Не меняйте:
   - `packet_id`
   - `conversation_id`
   - `blind_label`
   - `reviewer_id`
8. Сохраните файл как CSV UTF-8 и верните только заполненный `submission.csv`.

## Как оценивать

- `clarity`: насколько ответ понятен и хорошо структурирован
- `empathy_appropriateness`: уместность тона, уважительность, отсутствие грубости
- `usefulness`: насколько ответ помогает решить задачу пользователя
- `factuality`: насколько ответ выглядит фактически корректным и не вводит в заблуждение
- `robotic_repetition`: видно ли повторение штампов, одинаковых заходов или зеркалирование формулировки пользователя
- `unnatural_templating`: чувствуется ли чрезмерно протокольная, шаблонная или бюрократическая форма ответа
- `language_mismatch`: есть ли неуместный переход языка, смешение языков или ответ не на языке пользователя
- `clarification_helpfulness`: если ответ задаёт уточнение, помогает ли оно реально сдвинуть задачу вперёд
- `naturalness_feel`: насколько ответ в целом ощущается естественным человеческим текстом

## Правила blind review

- Не пытайтесь угадать, где `Helper`, а где `Baseline`.
- Не обсуждайте оценки с другими reviewers до завершения сбора.
- Оценивайте только по тексту в текущем пакете.
- Если ответ слабый, всё равно поставьте численную оценку, а не оставляйте поле пустым.
- Не оставляйте structured note columns пустыми; используйте разрешённые значения из списка выше.

## Формат ответа

Каждая строка в `submission.csv` соответствует одному blind-кандидату (`A` или `B`).
На один `conversation_id` должно быть две строки: одна для `A`, одна для `B`.
