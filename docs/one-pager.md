# HELPER One-Pager

## What It Is

HELPER is a local-first product shell for research, planning, generation, and operator-guided execution. It combines:

- a React UI
- the `Helper.Api` backend boundary
- the `Helper.Runtime` runtime

## What It Already Has

- passing release-baseline evidence
- a governed UI/API boundary
- local-first runtime separation through `HELPER_DATA_ROOT`
- evaluation and certification tooling
- parity-evidence scaffolding and traceability workflows

## What It Does Not Yet Prove

- human-level parity
- an authoritative completed blind human-eval round
- a completed counted `14-day` parity window

## Why It Matters

The project is designed to keep runtime state and secrets out of the repository, keep operator truth explicit, and make capability claims traceable to canonical evidence instead of narrative optimism.
