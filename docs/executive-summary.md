# HELPER Executive Summary

HELPER is a local-first operator shell for research, planning, generation, and runtime-guided execution. The public repository is a curated showcase that demonstrates product shape, honest status, and real generated artifacts without exposing the private-core implementation.

## What Exists Today

- a React-based operator UI
- a browser-facing `Helper.Api` boundary
- a `Helper.Runtime` backend runtime
- runtime/data separation through `HELPER_DATA_ROOT`
- real generated desktop artifacts captured from local demo runs
- release-baseline, certification, and evidence scaffolding

## Honest Status

- release baseline: `PASS`
- current certification state: `GREEN_ANCHOR_PENDING`
- human-level parity: `not proven`
- blind human evaluation: implemented, but the current corpus is still non-authoritative
- counted `14-day` parity window: not started

## Why It Matters

The project is built around controlled execution, traceable claims, and clear operator boundaries. That makes it a better fit for research workflows, planning workflows, generation workflows, and internal knowledge operations than a generic black-box chatbot.

## Repository Scope

This public repo is a showcase package, not the full runnable product. The private core, runtime data, internal scripts, and parity evidence bundle remain separate.
