# Status Definitions

This note defines the current public status terms used in the HELPER showcase repository.

## Why This Exists

The public repository uses a small set of shorthand status labels in `README.md`, `one-pager.md`, and `executive-summary.md`. This document explains what those labels mean in the public showcase context and, just as importantly, what they do not mean.

## Release Baseline: `PASS`

In the public showcase context, `Release baseline: PASS` means the current public pack is considered publishable as a sanitized showcase snapshot.

It does not mean:

- general availability
- full private-core certification
- human-level parity
- a claim that every private system surface is published here

## Current Certification State: `GREEN_ANCHOR_PENDING`

`GREEN_ANCHOR_PENDING` is the current internal certification-track label surfaced publicly for honesty.

In public terms, it means:

- no claim-eligible certification completion is being asserted here
- the project still has open certification and evidence work
- stronger certification claims should not be made from this repository

## Blind Human Evaluation: `implemented, but the current corpus is still non-authoritative`

This means the project has reviewer-intake and evaluation workflow scaffolding, but the currently available reviewer corpus is not yet strong enough to support headline performance claims.

In practical terms, `non-authoritative` means:

- the public project is not yet claiming a completed decisive reviewer corpus
- collection quality, volume, or governance are still incomplete
- the result should be treated as in-progress evidence, not final proof

## `14-day` Parity Window: `not started`

The counted `14-day` parity window is the planned fixed evaluation window for tracking parity-related evidence under stable rules.

`not started` means:

- that counted evaluation window has not begun
- no completed parity-window result is being claimed publicly
- the public repository should not be read as proof that such a window has already been completed

## Public Claim Boundary

These definitions are part of claim discipline for the public showcase.

The public repository is intended to communicate:

- product shape
- trust boundary
- current honest status
- a narrow runnable proof slice

It is not intended to stand in for the full private-core evidence bundle or a completed certification packet.

For the public method behind any future parity-related claim, see [parity-methodology-public.md](parity-methodology-public.md).
